# Известные баги

## Баг 1: Плавающий разрыв коннекции

**Статус:** Исследуется  
**Симптомы:** Коннекция периодически отваливается, UI показывает «Переподключаем…»  
**Вероятная причина:** Chrome убивает service worker в фоне (MV3 ограничение), переподключение ExtensionPort происходит не мгновенно.

**Предложения по исправлению:**
1. Client-side caching состояния в `chrome.storage.local` — показывать кэшированный чат при разрыве
2. Delayed UI update — не менять UI мгновенно при разрыве, подождать 1-2 сек (маскирует кратковременные разрывы)
3. Keep-alive пинг каждые 20 сек от sidepanel к background

## Баг 2: Индикатор работы агента не синхронизирован

**Статус:** Исследуется  
**Симптомы:** В консоли Pi видно «Working», субагенты работают, но в UI боковой панели индикатор не показывается. Создаётся впечатление, что Pi завис.

**Предложения по исправлению:**
1. Добавить логирование событий от Pi — проверить приходят ли `turn_start`, `tool_execution_start`
2. Если Pi не шлёт события при работе субагентов — создать issue в Pi SDK
3. Если события приходят, но не обрабатываются — исправить логику в `assistantState.ts`

**Отладка:** Добавлен лог в `backgroundStateServer.ts:260` — `console.log("[Pi Event]", event.type, event)`

## Баг 3: Неполная поддержка Markdown в UI чата

**Статус:** Зафиксирован  
**Симптомы:** Markdown рендерится частично. Текущая реализация (`markdown.ts`) поддерживает только:
- Блоки кода (```)
- Инлайн код (`)
- Жирный (**text**)
- Курсив (*text*)
- Ссылки ([text](url))
- Параграфы

**Не поддерживается:**
- Заголовки (#, ##, ###)
- Списки (-, *, 1. 2. 3.)
- Таблицы
- Цитаты (>)
- Горизонтальные линии (---)
- Подсветка синтаксиса кода
- Чекбоксы (- [ ] / - [x])

**Предложения по исправлению:**

### Вариант 1: marked + highlight.js (рекомендуется)
```bash
npm install marked highlight.js
```
- **Плюсы:** Полная поддержка GFM, подсветка синтаксиса, активно поддерживается
- **Минусы:** ~40KB gzipped (marked ~8KB + highlight.js ~30KB)
- **XSS:** Встроенная санитизация через `marked.setOptions({ sanitize: true })`

### Вариант 2: markdown-it
```bash
npm install markdown-it
```
- **Плюсы:** Модульный, можно подключать только нужные плагины, быстрый
- **Минусы:** Подсветка кода требует отдельного плагина
- **Размер:** ~12KB gzipped (без подсветки)

### Вариант 3: Расширить текущую реализацию
Добавить в `markdown.ts`:
- Заголовки: `/^(#{1,6})\s+(.+)$/gm`
- Списки: `/^[-*]\s+(.+)$/gm` и `/^\d+\.\s+(.+)$/gm`
- Цитаты: `/^>\s+(.+)$/gm`

- **Плюсы:** Минимальный размер, полный контроль
- **Минусы:** Много ручной работы, edge cases, нет подсветки кода
