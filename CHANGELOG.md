# Changelog

## [Unreleased]
### Added
- Popup расширения теперь разделён на вкладки `Асистент | Сессии | Авторизация`.
- Добавлено управление жизненным циклом browser token прямо в расширении: показ, копирование, перевыпуск и удаление.
- Добавлена команда `/chrome-assistent-auth` для авторизации браузера в Pi.

### Changed
- Публичная команда подключения переименована в `/chrome-assistent-connect`.
- Runtime-state Pi перенесён в `~/.pi/chrome-assistent/`.
- Browser trust теперь отделён от внутренней регистрации Pi-целей: браузер использует собственный browser token, а Pi — внутренний target token.
- Пользовательская документация переписана под flow `Авторизация → Сессии → Асистент`.

### Fixed
- Background и popup стабильнее обрабатывают browser-auth state и не дают устаревшим auth-ответам перетирать новый token.
- Браузерная аутентификация теперь не требует ручного редактирования хранилища через DevTools.

### Security
- Добавлен глобальный trusted browser store для авторизованных browser token.
- Browser token больше не совпадает с внутренним token регистрации Pi-целей.

## [2026-06-09] Browser Connect MVP review hardening
### Added
- Добавлены popup выбора цели, браузерная диагностика и DOM picker для отправки выбранного фрагмента страницы в Pi.
- Добавлена модульная документация по архитектуре, настройке, тестированию, устранению неполадок и модели безопасности MVP.

### Changed
- README переработан в короткую точку входа с быстрым стартом, ссылками на документацию и явными ограничениями MVP.
- Popup раньше блокирует отправку без ручной настройки токена и яснее показывает технические состояния MVP.

### Fixed
- Chrome-расширение теперь проходит `client.hello` перед получением списка целей и отправкой выделения.
- Pi-сессия восстанавливается после гонки запуска broker, если другой экземпляр уже успел занять порт `17345`.
- Улучшены выбор цели, диагностика popup и ограничения размера данных, отправляемых из DOM picker.

### Security
- `client.listTargets` больше не раскрывает локальные цели без предварительной аутентификации.
- Усилена защита токена Pi: файл и каталог проверяются без следования симлинкам и с жёсткими локальными правами доступа.

## [2026-06-08] Browser Connect MVP implementation
### Added
- Добавлена команда Pi `/chrome-assistent-connect [alias]` для подключения текущей сессии к локальному broker.
- Реализован localhost WebSocket broker для регистрации Pi-целей, heartbeat и доставки выделений в выбранную Pi-сессию.
- Добавлены типы протокола и форматирование браузерного выделения в сообщение, которое Pi получает в терминале.

### Changed
- Базовый пользовательский сценарий MVP построен вокруг локального broker на `127.0.0.1:17345` и чтения ответов Pi в терминале.

### Fixed
- Укреплены валидация протокола, обработка ошибок broker, таймауты доставки и очистка устаревших целей.
- Улучшены регистрация цели, подтверждение подключения и обработка переподключений Pi-сессии.

### Security
- Протокол отклоняет неизвестные и невалидные сообщения, а broker использует token для регистрации Pi-целей.
- Передаваемые в Pi текст и HTML ограничиваются по размеру, чтобы снизить риск чрезмерно больших отправок.
