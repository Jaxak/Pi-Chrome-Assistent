# Актуальная инструкция по подключению

## 1. Соберите расширение

```bash
npm install
npm run build:chrome
```

## 2. Загрузите расширение в Chrome

1. Откройте `chrome://extensions`.
2. Включите **Режим разработчика**.
3. Нажмите **Загрузить распакованное расширение**.
4. Выберите папку `dist/chrome`.

## 3. Запустите Pi

В корне проекта:

```bash
pi
```

Если локальное Pi-расширение не подхватилось сразу:

```text
/reload
```

## 4. Авторизуйте браузер

1. Нажмите на иконку расширения **Pi Chrome Assistent**.
2. Откройте вкладку **«Авторизация»**.
3. Скопируйте показанный токен браузера.
4. Внутри Pi выполните:

```text
/chrome-assistent-auth
```

5. Когда Pi запросит токен, вставьте токен из вкладки **«Авторизация»**.

## 5. Подключите текущую Pi-сессию

В нужной Pi-сессии выполните:

```text
/chrome-assistent-connect test
```

## 6. Выберите сессию в popup

1. Откройте вкладку **«Сессии»**.
2. Убедитесь, что цель `test` появилась в списке.
3. Выберите её.
4. При необходимости проверьте диагностику на этой же вкладке.

## 7. Отправьте фрагмент страницы

1. Перейдите на вкладку **«Асистент»**.
2. Нажмите **«Отправить в Pi»**.
3. Кликните по нужному элементу страницы.
4. Подтвердите отправку.

## 8. Где смотреть результат

- В браузере виден статус запуска picker и итог доставки.
- Ответ Pi нужно читать в терминале, где запущен `pi`.

## Если не работает

Проверьте:

- Pi всё ещё запущен;
- команда `/chrome-assistent-auth` была выполнена с тем же токеном, который показывает вкладка **«Авторизация»**;
- команда `/chrome-assistent-connect test` была выполнена в нужной сессии;
- цель выбрана на вкладке **«Сессии»**;
- расширение загружено именно из `dist/chrome`.
