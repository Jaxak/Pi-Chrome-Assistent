# Pi Chrome Assistent

## Описание проекта и текущие возможности

Pi Chrome Assistent — MVP-связка для Chrome-подобного браузера и [Pi Coding Agent](https://github.com/earendil-works/pi). Она позволяет выбрать активную сессию Pi, запустить DOM-пикер в текущей вкладке и отправить в Pi фрагмент страницы: URL, заголовок, CSS-селектор, текст, HTML и необязательный комментарий.

Что работает сейчас:
- браузерное расширение собирается в `dist/chrome` и загружается как неупакованное расширение Chrome;
- команда `/browser-connect [alias]` подключает текущую сессию Pi к локальному брокеру;
- popup-окно показывает доступные цели Pi и позволяет отправить выбранный фрагмент страницы;
- браузер получает статус доставки и диагностические сообщения.

Важно: в MVP ответы Pi не возвращаются в браузер и не стримятся в popup-окно — после отправки продолжайте читать ответ в терминале, где запущен `pi`.

## Быстрый старт

1. Убедитесь, что установлен Node.js `>=24`.
2. Установите зависимости и соберите расширение:
   ```bash
   npm install
   npm run build:chrome
   ```
3. Откройте `chrome://extensions`, включите режим разработчика и загрузите папку `dist/chrome` как неупакованное расширение.
4. Запустите `pi` в нужном проекте.
5. Выполните `/reload`, если локальное расширение Pi было добавлено после запуска `pi`.
6. Выполните `/browser-connect [alias]` в той сессии Pi, куда хотите отправлять данные из браузера.
7. Скопируйте общий токен из `.pi/browser-connect.token` и вручную сохраните его в `chrome.storage.local` под ключом `brokerToken`, например в DevTools сервис-воркера расширения:
   ```js
   await chrome.storage.local.set({ brokerToken: "<токен из .pi/browser-connect.token>" });
   ```
8. Откройте popup-окно расширения, выберите цель Pi, нажмите **Send to Pi**, затем выберите элемент на странице и при необходимости добавьте комментарий.

Под капотом браузер подключается к локальному брокеру `ws://127.0.0.1:17345`. Логи Pi-части пишутся в `.pi/browser-connect.log`.

## Текущие ограничения MVP

- Токен на стороне браузера настраивается только вручную; интерфейса для ввода или привязки пока нет.
- Браузер не показывает поток ответов Pi: есть только статус доставки и диагностика, а сами ответы остаются в терминале.
- Транспорт рассчитан только на локальный localhost-брокер `127.0.0.1:17345`.
- Жизненный цикл брокера в MVP зависит от активной сессии Pi с `/browser-connect`; после остановки такой сессии подключение из браузера может потребовать повторного запуска команды.
