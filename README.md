# Pi Chrome Assistent

Открывает боковую панель Chrome с чатом Pi и позволяет отправлять в активную сессию Pi как обычные сообщения, так и выбранный DOM-фрагмент страницы: URL, заголовок, текст, HTML-фрагмент и CSS-селектор.

---

## Установка

### 1. Установите Chrome-расширение

**Из Chrome Web Store:**
- [Pi Chrome Assistent](https://chromewebstore.google.com/detail/jpdbpapfbagejkgbkpfdabaogmhmheom) — нажмите «Добавить в Chrome»

**Или из пакета Pi (разработчикам):**
- Установите расширение в режиме разработчика из папки `dist/chrome` собранного пакета:
  1. Откройте `chrome://extensions`
  2. Включите **Режим разработчика**
  3. Нажмите **Загрузить распакованное расширение** → выберите `dist/chrome`

### 2. Установите пакет для Pi

```bash
pi install git:https://github.com/Jaxak/Pi-Chrome-Assistent.git
```

Если Pi уже запущен, перезагрузите расширения:
```
/reload
```

---

## Требования для сборки

Для сборки Chrome-расширения из исходников требуется:

- **Node.js** ≥ 24.0.0
- **rsvg-convert** (для генерации PNG-иконок из SVG)

### Установка rsvg-convert

| ОС | Команда |
|----|---------|
| Ubuntu/Debian | `sudo apt install librsvg2-bin` |
| macOS | `brew install librsvg` |
| Fedora | `sudo dnf install librsvg2-tools` |

## Быстрый старт

### 1. Авторизуйте браузер

1. Кликните по иконке **Pi Chrome Assistent** в панели браузера — откроется боковая панель.
2. В меню `⋯` выберите **«Авторизация»**.
3. Скопируйте показанный токен.
4. В терминале Pi выполните:
   ```
   /chrome-assistent-auth
   ```
5. Вставьте скопированный токен.

### 2. Подключите Pi-сессию

В терминале Pi выполните:
```
/chrome-assistent-connect
```

### 3. Выберите сессию в боковой панели

В блоке **«Сессия»** выберите нужную цель Pi из списка.

### 4. Напишите в чат

1. Введите текст в поле **«Сообщение ассистенту»**.
2. Нажмите **«Отправить»**.
3. Пока Pi отвечает, в чате отображается индикатор **«Агент работает в фоне…»** с тремя анимированными точками.
4. Ответ ассистента появится в боковой панели обычным текстом.

Первая версия чата намеренно не показывает tool calls и tool results — только пользовательские сообщения, текст ассистента, системные ошибки и индикатор фоновой работы.

### 5. Отправьте фрагмент страницы

1. В composer-меню `⋯` выберите **«DOM picker»**.
2. Наведите курсор на нужный элемент страницы — он подсветится
4. При необходимости уточните область:
   - **Меньше** — перейти к дочернему элементу (вглубь)
   - **Крупнее** — перейти к родительскому элементу (вверх)
   - **Вверх / Вниз** — перейти к предыдущему / следующему соседнему блоку
5. Кликните для фиксации, добавьте комментарий (необязательно) и нажмите **«Pi»**
6. Выделение будет отправлено в выбранную Pi-сессию; ответ также может появиться в боковой панели чата.

## Благодарности

- **DOM picker** — реализован с использованием решения [CrosshairJs](https://github.com/CodCatDev/CrosshairJs) для точного выбора DOM-элементов курсором.
