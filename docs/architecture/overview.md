# Обзор архитектуры

## Что это такое

Pi Chrome Assistent — это MVP-связка из трёх локальных частей:

1. **Расширение Pi** — добавляет команду `/browser-connect [alias]` в текущую сессию Pi.
2. **Локальный брокер** — WebSocket-сервер на `ws://127.0.0.1:17345`, через который общаются браузер и Pi.
3. **Chrome MV3-расширение** — popup, background service worker и content script для выбора DOM-фрагмента.

Схема обмена:

```text
Chrome popup / background / content script
                  │
                  ▼
        ws://127.0.0.1:17345
                  │
                  ▼
         Локальный browser broker
                  │
                  ▼
        Pi session A / B / C с /browser-connect
```

MVP отправляет данные **из браузера в Pi**. Ответ Pi остаётся в терминале, а в браузер возвращается только статус доставки и диагностическая информация.

## Основные роли компонентов

### Расширение Pi

- создаёт или читает общий токен из файла `.pi/browser-connect.token`;
- при необходимости поднимает локальный брокер;
- регистрирует текущую Pi-сессию как цель (`target`) с метаданными;
- получает выделенный браузерный фрагмент и передаёт его в `pi.sendUserMessage(...)`;
- пишет служебные логи в `.pi/browser-connect.log`.

### Локальный брокер

- хранит список активных целей Pi;
- принимает аутентификацию клиентов браузера по токену;
- маршрутизирует `client.sendSelection` в выбранную цель Pi;
- снимает с регистрации stale-цели, если heartbeat пропал.

### Chrome-расширение

- в popup показывает состояние брокера, список целей и диагностику;
- в background делает запросы к брокеру, хранит `brokerToken`, `selectedTargetId` и диагностику в `chrome.storage.local`;
- в content script запускает DOM picker, собирает полезный фрагмент страницы и отправляет его в background.

## Пользовательский поток: первое подключение

1. Пользователь запускает `pi` в нужном проекте.
2. При необходимости выполняет `/reload`.
3. Выполняет `/browser-connect [alias]`.
4. Pi читает или создаёт токен в `.pi/browser-connect.token`.
5. Pi пытается подключиться к брокеру на `127.0.0.1:17345`:
   - если брокер уже работает и токен совпадает, сессия присоединяется к нему;
   - если брокера нет, текущая Pi-сессия поднимает его сама;
   - если во время старта выясняется, что порт уже занял другой экземпляр, выполняется повторное подключение к уже поднятому брокеру.
6. Пользователь вручную копирует токен в `chrome.storage.local` браузерного расширения под ключом `brokerToken`.
7. Popup запрашивает список целей и даёт выбрать нужную Pi-сессию.

Подробности: [Pi-расширение](./pi-extension.md), [брокер](./broker.md), [ручная настройка токена](../operations/token-setup.md).

## Пользовательский поток: отправка фрагмента страницы

1. Пользователь открывает popup расширения.
2. Background обращается к брокеру:
   - открывает WebSocket;
   - выполняет `client.hello` с токеном;
   - запрашивает `client.listTargets`.
3. Popup показывает цели Pi и разрешает кнопку отправки только если:
   - есть хотя бы одна цель;
   - выбрана цель;
   - в браузере настроен `brokerToken`.
4. Пользователь нажимает **Send to Pi**.
5. Background внедряет `contentScript.js` в активную вкладку.
6. Content script включает режим DOM picker:
   - подсветка логического блока под курсором;
   - выбор по клику;
   - модальное окно с необязательным комментарием.
7. После подтверждения content script собирает payload:
   - `url`, `title`;
   - `selectedText`;
   - `selectedHtml`;
   - `selector`;
   - `comment`;
   - `capturedAt`.
8. Background снова открывает WebSocket, проходит `client.hello` и отправляет `client.sendSelection`.
9. Брокер доставляет payload выбранной цели Pi.
10. Pi форматирует сообщение и вызывает `pi.sendUserMessage(...)`:
    - сразу, если Pi простаивает;
    - как `followUp`, если Pi уже занят текущим ответом.
11. Браузер получает только результат доставки: успех или ошибка.

Подробности: [Chrome-расширение](./chrome-extension.md), [протокол](./protocol.md).

## Какие данные реально передаются

В Pi уходит текстовое сообщение с метаданными страницы и двумя основными блоками:

- выбранный текст — с ограничением `50 KiB`;
- HTML-фрагмент — с ограничением `100 KiB`.

Ограничения задаются в `src/shared/constants.ts` и применяются при форматировании сообщения для Pi.

## Что важно помнить про MVP

- Брокер слушает только `127.0.0.1`, а не внешний интерфейс.
- Токен на стороне браузера настраивается только вручную.
- Поток ответов Pi обратно в браузер не реализован.
- Жизненный цикл брокера зависит от Pi-сессии, которая его владеет.
- Из-за фиксированного порта и токена из `.pi/browser-connect.token` практический сценарий MVP — **один локальный broker-контур на один проект/токен**.
