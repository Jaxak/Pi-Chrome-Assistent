# Установка и запуск

## Требования

Перед началом убедитесь, что у вас есть:

- Node.js `>=24.0.0`;
- Chrome или совместимый Chromium-браузер;
- установленный и рабочий Pi Coding Agent.

## 1. Установите зависимости

```bash
npm install
```

## 2. Соберите Chrome-расширение

```bash
npm run build:chrome
```

После сборки папка `dist/chrome` должна содержать минимум:

- `manifest.json`
- `popup.html`
- `popup.css`
- `background.js`
- `popup.js`
- `contentScript.js`

## 3. Загрузите расширение в браузер

1. Откройте `chrome://extensions`.
2. Включите **Режим разработчика**.
3. Нажмите **Load unpacked** / **Загрузить распакованное расширение**.
4. Выберите папку `dist/chrome`.

## 4. Установите Pi-пакет и запустите Pi

Установите пакет:

```bash
pi install git:https://github.com/Jaxak/Pi-Chrome-Assistent.git
```

Пакет объявляет Pi-расширение в `package.json`, поэтому после установки Pi загружает `src/pi/browserConnectExtension.ts` автоматически. Откройте терминал в нужном проекте и запустите:

```bash
pi
```

Если пакет был установлен или обновлён при уже запущенном Pi, выполните:

```text
/reload
```

После загрузки должны быть доступны команды `/chrome-assistent-auth` и `/chrome-assistent-connect`.

## 5. Авторизуйте браузер через вкладку «Авторизация»

1. Откройте popup расширения.
2. Перейдите на вкладку **«Авторизация»**.
3. Скопируйте токен браузера.
4. В Pi выполните:

```text
/chrome-assistent-auth
```

5. Вставьте токен из popup в запрос Pi.

После этого браузер считается доверенным для локального runtime Pi.

## 6. Подключите нужную Pi-сессию

Внутри Pi выполните:

```text
/chrome-assistent-connect [alias]
```

Примеры:

```text
/chrome-assistent-connect
/chrome-assistent-connect frontend
/chrome-assistent-connect docs
```

Эта команда публикует текущую сессию как цель для браузера.

## 7. Выберите сессию на вкладке «Сессии»

1. Откройте popup.
2. Перейдите на вкладку **«Сессии»**.
3. Убедитесь, что нужная цель Pi появилась в списке.
4. Выберите её.
5. При необходимости посмотрите диагностику на этой же вкладке.

## 8. Отправьте фрагмент через вкладку «Асистент»

1. Перейдите на вкладку **«Асистент»**.
2. Нажмите **«Отправить в Pi»**.
3. Выберите элемент на странице.
4. При необходимости добавьте комментарий и подтвердите отправку.
5. Перейдите в терминал Pi и прочитайте ответ там.

## Полезные пути и адреса

- WebSocket broker: `ws://127.0.0.1:17345`
- Глобальный runtime Pi: `~/.pi/chrome-assistent/`
- Trusted browsers registry: `~/.pi/chrome-assistent/trusted-browsers.json`
- Логи Pi/broker: `~/.pi/chrome-assistent/chrome-assistent.log`
- Сборка Chrome-расширения: `dist/chrome`

## Что setup не делает автоматически

- не публикует Pi-сессию без `/chrome-assistent-connect`;
- не показывает ответы Pi в popup;
- не превращает broker в отдельный системный сервис вне жизненного цикла Pi.
