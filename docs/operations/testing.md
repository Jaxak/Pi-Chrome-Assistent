# Тестирование

## Автоматические проверки

Базовый набор команд проекта:

```bash
npm test
npm run typecheck
npm run build:chrome
```

Что они проверяют:

- `npm test` — unit/integration-тесты Vitest;
- `npm run typecheck` — проверка TypeScript без эмита;
- `npm run build:chrome` — сборка MV3-расширения в `dist/chrome`.

## Рекомендуемый порядок локальной проверки

1. Выполните:

   ```bash
   npm test
   npm run typecheck
   npm run build:chrome
   ```

2. Если всё прошло успешно, загрузите `dist/chrome` как unpacked extension.
3. Пройдите ручной smoke-сценарий ниже.

## Практический smoke-тест MVP

### Сценарий 1. Базовая отправка в одну Pi-сессию

1. Запустите `pi` в проекте.
2. Выполните `/reload`, если расширение Pi добавлялось после старта Pi.
3. Выполните `/browser-connect frontend`.
4. Скопируйте токен из `.pi/browser-connect.token` в `chrome.storage.local` как `brokerToken`.
5. Откройте popup расширения.
6. Убедитесь, что цель появилась в списке.
7. Нажмите **Send to Pi**.
8. Выберите элемент на странице.
9. Добавьте комментарий или оставьте поле пустым.
10. Подтвердите отправку.
11. Проверьте два результата:
    - в браузере появляется toast `Отправлено в Pi`;
    - в терминале Pi приходит сообщение с URL, текстом и HTML-фрагментом.

### Сценарий 2. Ранняя блокировка без токена

1. Удалите `brokerToken` из `chrome.storage.local`.
2. Откройте popup.
3. Проверьте, что:
   - кнопка отправки отключена;
   - popup показывает подсказку `Для отправки настройте brokerToken в chrome.storage.local.`;
   - DOM picker не стартует.

### Сценарий 3. Несколько Pi-сессий в одном проекте

1. Откройте две Pi-сессии в одном и том же проекте.
2. Выполните `/browser-connect` в каждой.
3. Откройте popup.
4. Убедитесь, что в списке есть две цели.
5. Выберите нужную и отправьте фрагмент.
6. Проверьте, что сообщение пришло именно в выбранную сессию.

### Сценарий 4. Потеря broker owner

1. Поднимите broker через `/browser-connect`.
2. Подключите вторую Pi-сессию к тому же broker.
3. Завершите сессию, которая владела broker server.
4. Проверьте, что после этого браузер может перестать видеть цели.
5. Повторно выполните `/browser-connect` в живой сессии и убедитесь, что связка восстановилась.

Это ожидаемое ограничение текущего MVP.

## Что удобно проверить вручную отдельно

- что при `Esc` DOM picker корректно отменяется;
- что при ошибке отправки показывается понятный toast;
- что `Diagnostics` в popup отражает последние ошибки;
- что `.pi/browser-connect.log` создаётся и пополняется;
- что после перезагрузки extension `brokerToken` и `selectedTargetId` сохраняются в `chrome.storage.local`.

## Полезные артефакты при отладке

- логи Pi/broker: `.pi/browser-connect.log`
- диагностика браузера: `chrome.storage.local["diagnostics"]`
- токен браузера: `chrome.storage.local["brokerToken"]`
- текущая выбранная цель: `chrome.storage.local["selectedTargetId"]`
