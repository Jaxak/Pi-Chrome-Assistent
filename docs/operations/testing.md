# Тестирование

## Автоматические проверки

Базовый набор команд проекта:

```bash
npm test
npm run typecheck
npm run build:chrome
```

## Рекомендуемый порядок локальной проверки

1. Выполните:

   ```bash
   npm test
   npm run typecheck
   npm run build:chrome
   ```

2. Если всё прошло успешно, загрузите `dist/chrome` как unpacked extension.
3. Пройдите ручной smoke-сценарий ниже.

## Практический smoke-тест

### Сценарий 1. Базовая отправка в одну Pi-сессию

1. Запустите `pi` в проекте.
2. Выполните `/reload`, если расширение Pi добавлялось после старта Pi.
3. Откройте popup, вкладку **«Авторизация»**.
4. Скопируйте token браузера.
5. Выполните `/chrome-assistent-auth` и вставьте этот token.
6. Выполните `/chrome-assistent-connect frontend`.
7. Откройте вкладку **«Сессии»** и убедитесь, что цель появилась в списке.
8. Выберите цель.
9. Перейдите на вкладку **«Асистент»**.
10. Нажмите **«Отправить в Pi»**.
11. Выберите элемент на странице.
12. При необходимости переключитесь по цепочке кандидатов кнопками **«Мельче»** и **«Крупнее»**.
13. Добавьте комментарий или оставьте поле пустым.
14. Подтвердите отправку.
15. Проверьте два результата:
    - в браузере появляется toast `Отправлено в Pi`;
    - в терминале Pi приходит сообщение с URL, текстом и HTML-фрагментом.

### Сценарий 2. Браузер ещё не авторизован

1. Перевыпустите token на вкладке **«Авторизация»**.
2. Не выполняйте `/chrome-assistent-auth` для нового token.
3. Откройте вкладку **«Сессии»**.
4. Проверьте, что browser получает ошибку авторизации.

### Сценарий 3. Несколько Pi-сессий в одном проекте

1. Откройте две Pi-сессии в одном проекте.
2. Выполните `/chrome-assistent-connect` в каждой.
3. Откройте вкладку **«Сессии»**.
4. Убедитесь, что в списке есть две цели.
5. Выберите нужную и отправьте фрагмент через вкладку **«Асистент»**.
6. Проверьте, что сообщение пришло именно в выбранную сессию.

### Сценарий 4. Потеря broker owner

1. Поднимите broker через `/chrome-assistent-connect`.
2. Подключите вторую Pi-сессию к тому же broker.
3. Завершите сессию, которая владела broker server.
4. Проверьте, что после этого браузер может перестать видеть цели.
5. Повторно выполните `/chrome-assistent-connect` в живой сессии и убедитесь, что связка восстановилась.

## Что удобно проверить вручную отдельно

- что при `Esc` DOM picker корректно отменяется;
- что на сложной web-app странице кнопки **«Мельче»** и **«Крупнее»** позволяют пройти по цепочке кандидатов вокруг элемента под курсором без потери текущего контекста;
- что picker не поднимается до выбора всего layout страницы без явной необходимости, если подходящий контейнер найден раньше;
- что popup и overlay действительно используют светлую оливковую тему, а контраст текста и кнопок остаётся читаемым;
- что диагностика остаётся на вкладке **«Сессии»**, а не на основном экране **«Асистент»**;
- что после перезагрузки extension `browserToken` и `selectedTargetId` сохраняются в `chrome.storage.local`;
- что `~/.pi/chrome-assistent/chrome-assistent.log` создаётся и пополняется.

## Полезные артефакты при отладке

- логи Pi/broker: `~/.pi/chrome-assistent/chrome-assistent.log`
- диагностика браузера: `chrome.storage.local["diagnostics"]`
- token браузера: `chrome.storage.local["browserToken"]`
- текущая выбранная цель: `chrome.storage.local["selectedTargetId"]`
