# Авторизация браузера

## Зачем нужна вкладка «Авторизация»

Теперь браузер не требует ручного редактирования `chrome.storage.local` через DevTools. Вместо этого popup показывает отдельную вкладку **«Авторизация»**, где можно:

- увидеть текущий токен браузера;
- скопировать его;
- сгенерировать новый токен;
- удалить токен.

Связка работает в два шага:

1. **Авторизовать браузер** через `/chrome-assistent-auth`.
2. **Подключить нужную Pi-сессию** через `/chrome-assistent-connect [alias]`.

## Базовый сценарий авторизации

1. Откройте popup расширения.
2. Перейдите на вкладку **«Авторизация»**.
3. Скопируйте токен браузера.
4. В Pi выполните:

```text
/chrome-assistent-auth
```

5. Вставьте токен из popup.

После этого Pi добавит токен в глобальный список доверенных браузеров.

## Как обновить токен

Если вы хотите перевыпустить токен браузера:

1. На вкладке **«Авторизация»** нажмите **«Сгенерировать новый токен»**.
2. Скопируйте новый токен.
3. Повторно выполните `/chrome-assistent-auth` в Pi и вставьте уже новый токен.

Важно: после смены токена старое разрешение перестаёт совпадать с браузером. Поэтому новый токен нужно авторизовать заново.

## Как удалить токен

На вкладке **«Авторизация»** нажмите **«Удалить токен»**.

Что это означает:

- текущий токен удаляется из браузерного хранилища;
- текущая вкладка popup показывает состояние без токена;
- при следующем открытии вкладки **«Авторизация»** расширение сможет создать новый токен заново.

Если вы удалили токен и создали новый, его снова нужно авторизовать через `/chrome-assistent-auth`.

## Что делать после авторизации

Авторизация браузера сама по себе не публикует Pi-сессию. После неё ещё нужно:

```text
/chrome-assistent-connect [alias]
```

Затем на вкладке **«Сессии»** выберите нужную цель Pi, а на вкладке **«Асистент»** отправьте фрагмент страницы.
