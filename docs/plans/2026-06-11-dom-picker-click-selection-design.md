# DOM Picker: Клик-фиксация выбора

> Дата: 2026-06-11
> Ветка: `feat/dom-picker-precision-and-olive-theme`

## Проблема

Кнопки «Мельче / Крупнее» находятся в панели overlay, на которую нужно навести мышку.
Перемещение мышки на панель разрывает hover на странице, и текущий выбор
перестаёт обновляться — пользователь не может отредактировать элемент,
который хочет отправить.

## Решение

Фиксация выбора по клику. После клика выбор замораживается и панель
управления появляется на экране. Mousemove по странице игнорируется,
пользователь свободно кликает по кнопкам панели.

## Состояния

### HOVER
- Пользователь водит мышкой по странице
- Рамка выделения: **1px** (тонкая, `1px solid #6f7f3a`)
- Панель управления: **скрыта** (`display: none`)
- Цепочка кандидатов обновляется при каждом `mousemove`
- **Клик на элементе** → переход в `SELECTED`

### SELECTED
- Выбор зафиксирован
- Рамка выделения: **2px** (яркая, `2px solid #6f7f3a`)
- Панель управления: **видима**
- Кнопки: `[Мельче] [Крупнее] [Изменить] [Отправить в Pi]`
  - «Отправить в Pi» — primary (оливковый фон)
  - Остальные — secondary
- `mousemove` по странице: **игнорируется**
- Клик по странице: **игнорируется**
- `[Мельче] / [Крупнее]` — меняют индекс в цепочке, рамка обновляется
- `[Изменить]` → возврат в `HOVER` (панель скрыта, рамка 1px)
- `[Отправить в Pi]` → модалка комментария → отправка → cleanup
- `[Отмена]` / `Escape` → cleanup, выход из picker

## Переходы

```
[START] → hover → click → [SELECTED]
  ↑                               │
  │     [Изменить]                │ [Отправить] → модалка → отправка → [END]
  └───────────────────────────────┘
                                  │
                            [Отмена]/Esc → [END]
```

## Изменения в коде

### `selectionOverlay.ts`

- Состояние `panelVisible: boolean`, панель по умолчанию скрыта
- `update(target: Element, selected?: boolean)` — при `selected=true` border 2px, иначе 1px
- Методы `showPanel()` / `hidePanel()` в `SelectionOverlayControls`
- Новый callback `onChange()` в `createSelectionOverlay`
- Кнопка «Изменить» (`data-testid="picker-change"`) между «Крупнее» и «Отправить в Pi»
- Сетка кнопок: `grid-template-columns: repeat(4, 1fr)`

### `contentScript.ts`

- State: `'hover'` | `'selected'`
- `mousemove` — работает только в `'hover'`
- Новый `click` handler (capture phase):
  - В `'hover'` → фиксирует элемент, `overlay.update(el, true)`, `overlay.showPanel()`, состояние → `'selected'`
  - В `'selected'` → игнорируется
- `onNarrow`/`onWiden` — работают в обоих состояниях
- `onChange` (новая кнопка) → состояние → `'hover'`, `overlay.hidePanel()`, `overlay.update(currentEl, false)`

### Тесты

**`contentScript.test.ts`:**
- Клик фиксирует выбор и показывает панель
- `mousemove` игнорируется после фиксации
- `onChange` возвращает в режим hover
- `onNarrow`/`onWiden` работают с зафиксированным выбором

**`selectionOverlay.test.ts`:**
- Панель скрыта по умолчанию
- `showPanel()` / `hidePanel()` переключают видимость
- Кнопка «Изменить» присутствует с русским текстом
- Рамка 1px по умолчанию, 2px при `selected=true`

## Граничные случаи

- Клик на UI picker-а — `isPickerUiElement` защищает
- `onChange` → hover → быстрый клик — работает без задержек
- Элемент без кандидатов — минимально `[hovered]`, не ломается
